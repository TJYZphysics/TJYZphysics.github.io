# 光路寻踪与主页博文转盘设计

## 目标

将实验区的“因果折纸”重构为“光路寻踪”，提供不少于 20 个由易到难、始终可以直接访问且全部可解的关卡；加入分光棱镜等可组合道具。同步修复主页标题的中文字距与行距，并把单篇硬编码博文推荐改为转盘式、边缘渐隐的动态推荐。

## 全局约束

- 网站继续保持纯静态 GitHub Pages 架构，不新增后端、账号或数据库。
- 关卡不锁定、不保存进度，用户可以随时访问任意关卡。
- 至少提供 20 个关卡，每关必须保存一条完整标准解，并由测试通过玩家相同的放置接口重放验证。
- 验收时除单元测试和构建外，必须在浏览器中人工检查页面，并逐关运行标准解确认成功。
- 保持现有深色科学实验视觉语言，兼顾桌面和移动端。

## 光路寻踪

### 玩法模型

棋盘保持统一的 9 列、8 个时间步，降低关卡切换时的布局风险。两束初始光从棋盘顶部进入，每经过一个时间步横向移动一格。玩家在事件点放置道具，引导有效光线避开禁区并在目标事件会合。

道具采用判别联合类型：

- `turn-left`：将经过该点的光线方向改为向左。
- `turn-right`：将经过该点的光线方向改为向右。
- `splitter`：保留入射方向，并额外产生一条反向分支。

每关分别定义三类道具库存。放置点不能是起点、目标、禁区或棋盘边界之外；同一事件点只能放置一个道具。模拟器按时间步维护有界光线分支，使用来源、位置、时间和方向去重，防止分光产生重复状态。

胜利条件为：至少一条来自每个初始光源的合法分支同时抵达目标事件。任何抵达目标的分支即完成使命；其他分支越界或进入禁区不会抹除已经存在的合法目标分支，但若某个光源没有合法分支抵达目标则不能通关。

### 关卡结构

每个关卡包含：唯一 ID、序号、名称、难度、教学提示、棋盘尺寸、两个光源、目标、禁区、道具库存和完整标准解。

20 关按以下节奏递进：

1. 第 1–4 关：单次或对称偏转，学习左右偏转器。
2. 第 5–8 关：增加预算限制和禁区绕行。
3. 第 9–12 关：引入分光棱镜及单侧分支选择。
4. 第 13–16 关：偏转器与分光棱镜组合，加入干扰路径。
5. 第 17–20 关：紧预算、多禁区和非对称组合题。

关卡选择器始终展示全部 20 关，可直接点击切换。切关时清空当前放置、运行结果和撤销历史。上一关、下一关按钮用于快速浏览，但不承担解锁逻辑。

### 可解性保证

关卡采用“答案先行”设计：先写标准解并确认路径，再加入不破坏标准解的禁区和干扰元素。测试通过公开的 `placeTool` 接口逐一放置标准解，然后调用与 UI 相同的 `runLevel`。

测试对每关断言：

- 总数不少于 20，ID 和序号唯一。
- 所有坐标合法，目标时间等于关卡持续时间。
- 标准解不落在起点、目标或禁区。
- 标准解使用量不超过各道具库存。
- 每一步放置均成功。
- 模拟结果合法且成功，两方光源均有分支抵达目标。

### 界面

标题改为“光路寻踪”，副标题强调“布置镜片与棱镜，让两束光在目标事件会合”。顶部增加关卡编号、难度和直接选择关卡的面板。工具栏使用三张清晰的工具按钮展示图标、名称和剩余数量；分光棱镜使用紫色菱形标识，左右偏转器保持青色箭头语义。

棋盘继续用 SVG 绘制光路，但支持多条分支。不同初始光源使用青色与紫色，分支继承来源颜色并降低次级分支透明度。运行成功后提供醒目的绿色反馈和“下一关”快捷按钮。

## 主页标题

标题显式拆成两个块级行：

- 第一行：“在已知的边界”
- 第二行：“继续发问。”，其中“发问。”保留描边强调

移除中文不适合的负字距，桌面使用轻微正字距和约 1.04 行高；移动端降低字号并增加行高，保证 360px 宽度下不溢出、不重叠。

## 主页博文转盘

主页直接复用 `getBlogPosts()`，不再硬编码文章。最多展示最近 6 篇，当前文章位于中央并保持完整可读；前后文章沿浅弧轨道缩小、降低透明度并露出边缘。

转盘容器使用 CSS `mask-image` 和 `-webkit-mask-image` 制作左右边缘渐隐。提供左右箭头、分页圆点、当前序号和键盘方向键。文章数为 0 时显示空状态；1 篇时静态展示；2 篇以上循环轮换。自动轮换间隔为 7 秒，鼠标悬停、键盘聚焦或页面不可见时暂停；`prefers-reduced-motion` 下关闭自动播放和位移动画。

没有封面图时，卡片根据文章索引与标签生成稳定的深蓝、青色或紫色径向渐变，不新增图片资源。移动端只完整展示当前卡片，并让相邻卡片少量露边，控制按钮移到卡片下方。

## 测试与验收

- 模型测试覆盖三类道具、库存、非法放置、分光去重和成功条件。
- 关卡测试重放全部标准解，作为“20 关全部有解”的持续保证。
- 组件测试覆盖任意关卡选择、切关清空状态、文章数据渲染与轮播控制。
- 执行 `npm test` 和 `npm run build`。
- 使用浏览器人工检查桌面与窄屏主页，对比改动前后截图。
- 在浏览器中依次选择 20 关，按标准解放置道具并运行，确认每关显示通关成功。
